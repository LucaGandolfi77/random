# Atlas Trial Mode

## Overview

This skill puts the work on trial before it ships. Use it when the assignment is too sensitive, exposed, or contested for an ordinary review pass.

Trial Mode is not brainstorming. It is not open-ended critique. It is a structured adversarial hearing with defined roles and a verdict.

## Roles

1. `fact_prosecutor`: builds the strongest admissible case against publication.
2. `defense_editor`: builds the strongest honest case for what remains salvageable.
3. `trial_judge`: weighs both sides and issues the ruling.
4. `legal_reviewer`: identifies claim-boundary and exposure risk.
5. `copy_chief`: normalizes the release if the verdict allows publication after fixes.
6. `audience_editor`: tests whether a public reader could still trust the piece.
7. `continuity_archivist`: checks that facts, prior versions, or canon do not contradict the draft.

## When to Use

- The work is politically sensitive, reputation-sensitive, or legally exposed.
- The piece contains serious allegations, operational instructions, or high-risk claims.
- The user wants an adversarial review, not a friendly polish pass.
- The release decision is materially contested.

Do not use this skill for:

- casual drafting
- ordinary low-risk copy editing
- first-pass ideation
- theatrical argument without evidence

## Workflow

1. State the artifact and the release question.
2. Assign prosecutor, defender, and judge.
3. Add legal reviewer, audience editor, copy chief, or continuity archivist only when justified.
4. Present the prosecution case first.
5. Present the defense case second.
6. Present legal-risk notes when relevant.
7. Issue one verdict: `Publish`, `Publish after fixes`, or `Do not publish`.
8. Name the smallest fix set that would change the ruling.

## Output Pattern

1. `Artifact:` what is on trial
2. `Court:` prosecutor, defender, judge, additional officers
3. `Charges:` the case against publication
4. `Defense:` the strongest honest salvage case
5. `Legal notes:` only when relevant
6. `Verdict:` Publish / Publish after fixes / Do not publish
7. `Smallest fix set:` the exact changes that would alter the verdict

## Decision Standard

- The prosecutor must argue the strongest admissible case, not a theatrical one.
- The defense must concede real defects and argue only what can honestly survive.
- The judge must reject style points and rule on evidence, structure, clarity, risk, and release readiness.

## Verification Checklist

- [ ] Prosecutor, defender, and judge are named explicitly.
- [ ] The prosecution case precedes the defense case.
- [ ] The verdict is explicit.
- [ ] The smallest fix set is concrete.
- [ ] The review stays in English and avoids literary imitation.
